# Recursos abertos - CV Aprovado

Pacote inicial publicado como repositorio aberto para apoiar aquisicao organica e respostas uteis em comunidades.

Objetivo:

- gerar trafego organico indexavel;
- dar material util para candidatos e recrutadores;
- criar links de apoio para Reddit, Substack, Discord, Slack e comunidade WhatsApp;
- reduzir risco de spam em comunidades usando conteudo pratico antes de qualquer oferta comercial.

Link principal:

```text
https://www.cvaprovado.com/?utm_source=github&utm_medium=open_resource&utm_campaign=organic_acquisition_2026
```

## Recursos prontos

- `checklist-ats.md`: checklist simples para revisar um curriculo antes de aplicar.
- `checklist-entrevista-triagem.md`: preparacao para entrevista inicial com recrutador.
- `checklist-linkedin.md`: checklist para deixar o LinkedIn mais claro para recrutadores.
- `como-pedir-feedback-de-cv.md`: guia para pedir feedback publico sem expor dados pessoais.
- `exemplo-resumo-profissional.md`: exemplos de resumo profissional menos generico.
- `indice-links-comunidade.md`: links diretos por tema para respostas em comunidades.
- `LICENCA-USO.md`: permissao de uso, atribuicao e limites de compartilhamento.
- `template-vaga-verificada.md`: modelo de vaga para recrutadores publicarem na comunidade.
- `guia-anti-golpe-vagas.md`: sinais de alerta para candidatos evitarem vagas falsas ou abusivas.
- `matriz-de-aderencia-vaga.md`: matriz para comparar requisitos da vaga com evidencias reais.
- `mensagens-para-pedir-indicacao.md`: modelos para pedir indicacao com contexto e respeito.
- `modelo-curriculo-anonimizado.md`: modelo seguro para pedir feedback em comunidade.
- `perguntas-para-recrutadores.md`: perguntas para entender vaga, processo e riscos antes de aplicar.
- `plano-repo-publico.md`: plano de publicacao, descricao do repo e proximos arquivos.
- `roteiro-plantao-curriculo.md`: roteiro para organizar revisoes de curriculo em comunidade.

## Posicionamento

O CV Aprovado nao promete emprego, contratacao ou "passar no ATS". A proposta e melhorar clareza, aderencia e seguranca na busca por vagas.

## Regras de uso em comunidades

- Nao usar o material para responder em massa com texto repetido.
- Nao postar link como primeira interacao quando a comunidade nao permitir divulgacao.
- Nao pedir dados pessoais ou curriculos completos em comentarios publicos.
- Usar exemplos anonimizados e orientar a pessoa a remover nome, telefone, endereco, CPF e links privados antes de pedir feedback publico.
- Preferir links especificos do `indice-links-comunidade.md` em vez de mandar apenas a pagina inicial do repo.
